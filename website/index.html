Background

Describe the problem that you are trying to solve: The biggest advantage of using ray tracing over rasterization is that ray tracing The biggest disadvantage of ray tracing is it's speed and efficiency compared to raterization. The problem I will try to solve is the speed of our ray tracer. This is a challenging feat because not all rays will have the same direction and origin. So we must first find similar rays, which would be difficult if we wanted to use Whitted-style ray tracing (WRT), which adds reflection and refraction as we did in the 3rd assignment. If we were to only Ray cast with shadows (RCS) we could use a common origin to find similar rays and create our packets. But when a bundle of rays in a packet hit a material and refract/reflects, such as in WRT, the origin and direction of the rays will be dissimilar and not share a common origin. This is an important issue to computer graphics because to physically render realistic images we need to use WRT over RCS. But the computational cost of WRT is too much and not feasibly on most computers without a graphics card. This could be seen with the assignment 3/4. I personally care about this because of feeling I got when I first rendered CBSphere.dae in project 3 with 1024 ssp and 100 bounces. I was so impressed with the visuals but I was disappointed with the time it took my machine to render the image. Some resources that I have been looking into to improve our pathtracer include https://graphics.stanford.edu/~boulos/papers/cook_gi07.pdf and http://graphics.ucsd.edu/~henrik/papers/coherent_path_tracing.pdf. Both of these papers discuss the problem of secondary rays, rays that reflect/refract.
